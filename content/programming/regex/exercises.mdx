## Common Patterns Reference

Here are some patterns worth memorizing:

| Task | Pattern |
|---|---|
| One or more digits | `\d+` |
| Optional sign + number | `[+-]?\d+` |
| Decimal number | `\d+\.?\d*` |
| Word (no punctuation) | `\b[a-zA-Z]+\b` |
| Whole line match | `^.*$` |
| Empty line | `^$` |
| Repeated word | `\b(\w+)\s+\1\b` |
| Hex color | `#[0-9A-Fa-f]{3,6}` |
| ISO date | `\d{4}-\d{2}-\d{2}` |
| Simple email | `[\w.+\-]+@[\w\-]+\.[a-zA-Z]{2,}` |

## Next Steps

You now know everything needed to use regex confidently in real projects. Some directions to explore next:

- **Regex in your language** — learn the specific API for Python (`re` module), JavaScript (built-in), Go (`regexp` package), or Rust (`regex` crate)
- **Performance** — learn about possessive quantifiers and atomic groups to prevent catastrophic backtracking
- **Unicode regex** — use `\p{L}`, `\p{N}` and Unicode categories for international text
- **Regex in tools** — `grep`, `sed`, `awk`, VS Code search, and database queries all support regex
